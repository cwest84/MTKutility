The Things You Need to Know

MTKutility is a maintenance app for Bluetooth capable GPS receivers and loggers that use the MTK (GPS) chip.  The MTK GPS devices that it will work with are GPS receivers (no logging) with or without AGPS and GPS loggers with or without AGPS.  It has been tested on physical phones running Android 4.1, 4.4, 5.1 and 8.1.

MTKutility uses Android's Navigation Drawer menu. To show the menu click on the 3 bars at the top left of the screen, or alternatively slide your finger to the right from the left edge of your screen.  The menu will only show actions that your GPS device supports.  You have to connect to your GPS device before the full menu for your GPS device will show.

The Home and Update Settings screen font sizes are configurable. They have been set for an 720x1280 pixel device.  You may need to adjust font size for your phone to improve your experience.  Click the wrench symbol at the top right to open the preferences screen, click the font you want to adjust and enter a new size, and refresh the screen by reselecting it in the navigation menu.  Please email if you need any of the other screens adjusted for your phone.

Home screen

This screen has five areas.  The top (1st) box shows the AGPS data expiry date on the first line and the number of GPS track points stored on the second line when the GPS device is connected.  The 2nd box is for NMEA settings. The 3rd (unboxed) area buttons send commands to your GPS device.  The 4th box is a scrollable message area for important events,  The 5th box is the scrollable NMEA sentence display area.

NMEA (National Marine Electronics Association) is a combined electrical and data specification for communication with GPS receivers.  MTKutility provides for setting the display frequency of 7 formats:


 GGA Global Positioning System Fix Data

GLL Geographic position, Latitude and Longitude

GSA Satellite status

GSV Satellites in view

RMC Recommended Minimum sentence Content

VTG Vector track an Speed over the Ground

ZDA Date and Time

The row with these values scrolls left and right. Each has a selection pull down with values 0 through 5 that determine if the format is displayed and how frequently it is displayed.  The Save Settings button will only work when you are connected to a GPS device.  Your setting choices are:


 0 Disabled

1 Output once every one position fix

2 Output once every two position fixes

3 Output once every three position fixes

4 Output once every four position fixes

5 Output once every five position fixes

Your GPS device has to be paired in your phones Bluetooth settings before you can select it with the set Logger button. The selection is automatically saved.  Use the Connect button to establish a Bluetooth connection with your GPS device.  Try the allow insecure connect if you experience difficulty connecting.

The NMEA sentences sent by your GPS device scroll in the bottom part of the screen after you connect.  MTKutility is not a GPS track recorder. The NMEA output confirms an active Bluetooth connection and shows what is transmitted if you are using you logger as the GPS data source for another device.

Use the restart buttons to reset your logger.


 hot reset means use all available data in the NV Store

warm reset don't use Ephemeris at re-start

cold reset don't use Time, Position, Almanacs and Ephemeris data at re-start

factory reset cold restart and clear system/user configurations (reset the receiver to the factory status).

Download Log screen

Use the Download Log button to download the track point data from your GPS logger.  The file created is a raw GPS logger memory dump.  It cannot be used directly by track display apps.  The file is created in your phone's download folder in the sub-folder MTKutility/bin.  The storage location is arbitrary.  You can transfer the binary file to another location using the phone's File Manager app.  The Erase GPS Records button triggers a format of the track point storage space on your logger.

The log download has three setable preferences: Disable NMEA when downloading, Disable GPS logging when downloading and Download Block Size.  Disable NMEA when downloading is turned on by default.  Play with these if your download isn't happening.

Make GPX file screen

Use this screen to create GPX files from previously downloaded binary files.  You do not need to connect to your GPS logger to use this.  You have to select the binary file you want to decode before you can create the GPX file.  The file browser defaults to the MTKutility/bin location but you can navigate up to other locations.  You should be able to read a file from any user accessible storage location on your phone including your SD card.  Your phone's security will determine what you can do.

The file browser has been designed to let you navigate folders by clicking folder names.  Click the file icon to select a file.

NOTE: The GPX file should have enough data for geotagging pictures but it may not have all the data you selected for the log.  Use BT747 to create your GPX files if you use more than the default settings for your logger.

Get EPO file screen

Use this screen to down load an Extended Prediction Orbit (EPO) file for your AGPS data update.  There are 2 flavors of EPO data, 60 byte data SET and 72 byte data SET.  the Select FTP URL button currently has 2 sites available: the Qstarz 60 byte site and the Globaltop 72 byte site.  You will need to rely on trail and error if you do not know which site to pick.  My Qstarz logger will accept 72 byte records without error but it does not store the EPO data - and it takes forever (hours) to complete.  There is also a configuration item associated with AGPS updates.  Your logger will have a maximum size for stored records. I believe most loggers store 7 days but the sure way to know is update the AGPS using your PC application.  Then note the EPO set value on the Home screen after you connect, divide this by 4, and set the result in your preferences.

The Globaltop 72 byte site has 10 files, each with 3 days EPO data to make up a 30 day set.  Updating with successive files will only result in 3 days data being stored on your GPS.  You have to join files to get a 7, 14 or 30 day AGPS update.  Here's how.  Download the EPO_GPS_3_1.DAT file.  Then click the Select Local File button and choose this file. It will appear in the local file name slot and is the file you will append to.  Then click the Select EPO File button again and select the EPO_GPS_3_2.DAT file.  This will unlock the Append EPO file button.  Click it to do the append.  Repeat with the Select EPO File - Append EPO file buttons combination until you have enough days collected to do a full AGPS update.

The Globaltop 72 byte site also has 60 byte files (MTK7d.EPO, etc.) but the last update I see was Jan 16, 2019.

Update AGPS screen

You need to select the EPO data you plan to use before you can run the update.  It is also a good idea to download first as EPO files are updated each day.  Click Select EPO File button.  The default MTKutility/bin directory will open but you can navigate to a different location if your file is not in this folder.  Click the file icon to select the file.  This unlocks the Update AGPS data button.  MTKutility will limit the AGPS update to the size limit set in configuration.  You can use a 30 day file for a 7 day update without having to worry about extra processing time for the larger file.

Update Settings screen

The settings screen has a cross section of GPS settings - not all possible settings but all that directly affect logging.  It is made up of a set of buttons and an HTML report of your GPS device settings. The button rows match the rows in the HTML table.

The Method button has radio buttons for what happens when the logger's recording when memory is full.  Leave this set at Stop if you want to use MTKutility to download trackpoint data.  I do not have enough inormation to implement download for Overwrite.

The only other button that doesn't show check boxes is the Frequency button.  The by distance and speed over values in the popup have to be whole numbers.  The by time setting can be in tenths of a second - as fast as every tenth of a second (0.1).  Works well for sky divers.  But a by time setting under 1 second is not saved when you turn the logger off.  The logger reverts to its default value. The logger capacity information is based on a memory size vlaue in device information.  It may not be correct for your logger.  The bitmask at the very bottom is only useful if you are intimately familiar with how the settings are stored.

This screen has 2 size settings: one for the buttons and one for the HTML table.  Adjust them in preferences (wrench at top right and reselect screen after changing) so that everything fits comfortably on your phone's screen.

Send App Log screen

Use this to send a trace log by email to me.  You need to choose either the MTKutilityLog.txt or MTKutilityErr.txt first.  The MTKutilityErr.txt file is created when MTKutility encounters an unrecoverable error.  The app is coded to catch these errors and restart itself with a popup to send the file to me.  Please do so when this happens as this usually reflects a problem that has to be fixed.

The MTKutilityLog.txt file will give me information about your phone and, if you connect before sending, the GPS device's information string.  Please send a file to me at least once.  Knowing device combinations will help with me with problem resolution and future enhancements.  There is no personal information in the file and you can verify this by looking at a file before you send it.

crash reporting

All MTKutility public files are written to your phone's downloads folder in a sub-folder named MTKutility.  The execution trace log is created in a file named MTKutilityLog.txt.  A file named MTKutilityErr.txt is created when MTKutility crashes. MTKutility automatically restarts when a crash happens and presents the option to email the MTKutilityErr.txt file. Please email this file to me. If you can, add a short description of what you were trying to do when MTKutility crashed. My ultimate goal is to have MTKutility work reliably with all MTK chip GPS devices.

will not work with my GPS logger

email me if MTKutility does not work with your phone and GPS combination.  Please turn the verbose level 3 log trace level on and try to repeat the error, then send the report to me.  Use the menu Send App Log function to send the trace log if the app doesn't automatically restart.  Please add the make and model of both your phone and GPS as information as the trace log may not have captured this.




dealing with problems that arise
Your MTK logger is put into binary communication mode for the AGPS data update.  If MTKutility fails during the AGPS update, your logger will remain in binary mode and will not function as a track point recorder.  Make sure you run MTKutility again to restore NMEA output mode if it fails during an AGPS update.  MTKutility may be your only means of correcting this.  My Qstarz PC software doesn't.

If you selected the Disable NMEA when downloading preference, a crash while downloading may result in NMEA output being turned off - no scrolling display on the  Home screen when you connect.  Use the Default NMEA button on the Home screen to restore output.

If you selected the Disable GPS logging when downloading preference, you will need to turn your logger off after downloading to restore trackpoint recording capability.  This can't be restored by the app as it overrides the manual setting on loggers that act as either a trackpoint transmitter of trackpoint recorder.

Both the log download and AGPS update do occasionally stop before completion.  This is usually the result of a Bluetooth communication interruption.  MTkutility has screen rotation turned off and display set to portrait to try an mitigate this but it can still happen.   The only recovery I've been able to find for this is to turn my GPS off and restart my phone.

Actions to take if MTKutility crashes: cycle your phone's Bluetooth off and on, turn your GPS off and on, and try connecting again.  MTKutility pushes Android Bluetooth capability and implements data exchanges that would be better handled via a USB connection.  However, I have not been able to get the download to work over an OTG connection so Bluetooth is my only option.
